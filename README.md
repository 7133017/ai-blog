# AI-Blog
一个极简、免数据库、开箱即用的PHP博客系统，轻量到只有单个PHP文件，却能满足个人博客的核心需求 ✨

## 特性
- 📦 单文件部署：整个系统只有一个PHP文件，无需配置数据库
- 🔒 安全防护：CSRF令牌、登录尝试限制、路径校验
- 📝 核心功能：文章发布/编辑/删除、标签分类、分页展示
- 🗄️ 轻量存储：文章以JSON+Markdown文件存储，无需数据库
- 🚀 缓存优化：文章列表缓存，提升访问速度
- 🎨 响应式设计：适配手机/桌面端，极简美观的默认样式
- 🔐 自定义登录路径：避免默认后台路径被扫描

## 快速开始
### 环境要求
- PHP 7.2+
- 服务器需有读写权限（data目录自动创建）

### 部署步骤
1. 下载 `index.php` 文件到你的Web服务器目录
2. 访问服务器域名/IP，自动进入初始化页面
3. 填写站点名称、管理员账号密码、自定义登录路径（留空自动生成随机路径）
4. 保存后自动跳转到登录页面，登录即可开始写博客

## 使用说明
### 核心操作
- 登录：访问 `?a=你的登录路径`
- 写文章：登录后点击「写文章」，支持Markdown格式（前端暂未渲染，可自行扩展）
- 标签管理：用逗号分隔多个标签，点击标签可筛选对应文章
- 文章管理：支持编辑/删除、公开/私密设置、自定义发布时间

### 目录结构
```
├── index.php          # 主程序文件
└── data/              # 自动生成的存储目录
    ├── config.php     # 站点配置文件（初始化后生成）
    ├── posts/         # 文章存储目录
    │   ├── {id}.md    # 文章内容
    │   └── {id}.meta.json # 文章元数据（标题/标签/时间等）
    └── cache/         # 缓存目录
        └── posts_cache.json # 文章列表缓存
```

## 自定义扩展
- 修改默认样式：直接编辑PHP文件中的`<style>`部分
- 支持Markdown渲染：引入Markdown解析库（如Parsedown）替换`nl2br(htmlspecialchars())`
- 增加搜索功能：现有代码已预留搜索参数，可扩展前端搜索框
- 自定义页面：通过新增路由分支扩展About/Archive等页面

## 安全提示
- 定期备份 `data` 目录，包含所有文章和配置
- 不要将 `data` 目录暴露在Web访问路径下（默认已通过路径校验防护）
- 建议设置强密码，并定期更换
- 登录路径建议使用随机字符串，避免简单路径
